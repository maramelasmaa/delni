## Architecture

### Service Worker Strategy

The service worker (`resources/js/sw.js`) implements **explicit route allowlisting** to protect sensitive pages:

#### ✅ Cached (Public Marketplace Only)
- Static Vite assets (JS, CSS, fonts) — **CacheFirst** (30 days)
- Provider images — **StaleWhileRevalidate** (7 days)
- Public marketplace pages — **NetworkFirst** (10 min TTL):
  - Homepage, search, categories, providers, top-rated, etc.
- API search requests — **NetworkFirst** (5 min TTL)

#### ❌ Never Cached (Network Only)
- Admin panel (`/cp/*`)
- Provider panel (`/provider/*`)
- Auth pages (`/login`, `/register`, `/logout`)
- Account pages (`/settings`, `/dashboard`, `/favorites`)
- Password reset (`/reset-password/*`)
- Onboarding (`/onboarding/*`)
- All POST/PUT/DELETE requests

### Service Worker Registration Gating

The service worker only registers on public pages via route checks in `resources/views/public/layout.blade.php`:

```php
$shouldRegisterPublicPwa = request()->routeIs(
    'home',
    'public.search',
    'public.top-rated',
    // ... other public routes
);
```

Service worker registration code only executes if `$shouldRegisterPublicPwa` is true:

```js
@if($shouldRegisterPublicPwa)
if ('serviceWorker' in navigator && window.isSecureContext) {
    navigator.serviceWorker.register('/sw.js', { scope: '/' }).catch(() => {});
}
@endif
```

## Marketplace Safety

### Visibility Rules Respected

The PWA uses **NetworkFirst** strategy for dynamic marketplace pages, ensuring:

- Suspended providers disappear quickly (cache timeout 10 min max)
- Expired subscriptions removed (cache timeout 10 min max)
- Featured placements update quickly
- Incomplete profiles hidden
- Visibility rules enforced on every network request

**No stale provider visibility.**

### Cache Invalidation

Static assets (Vite hashes) are immutable and cache-friendly (30 days).  
Dynamic provider data uses short TTLs and NetworkFirst strategy to respect backend visibility.

## Android Support

Android PWA installation uses the manifest:

- `display: standalone` — full app experience
- `theme_color`, `background_color` — app shell branding
- App icons for home screen
- Safe area support via viewport-fit

## Offline Experience

When offline or after cache expiry, users see:

1. Cached marketplace pages (if available)
2. `public/offline.html` fallback (Arabic, branded)

Offline banner shows at top when connection lost:

```
أنت غير متصل بالإنترنت حاليا. بعض بيانات مقدمي الخدمات قد لا تكون محدثة.
```

## Cache Strategy Verification

In DevTools → Application → Cache storage:

- `vite-assets` — Vite JS/CSS/fonts (30 days)
- `provider-images` — Profile images (7 days)
- `api-search` — Search API (5 min)
- `marketplace-pages` — Public pages (10 min)
- `network-only` — Admin/auth (network only)
- `offline-fallback` — offline.html

Each cache should have appropriate TTL and content.

## Known Limitations

- Offline search doesn't work (requires API call)
- Admin panels never cached (secure)
- Auth pages never cached (secure)
- Marketplace visibility rules enforced on every network request
